news-pipeline-run 스킬을 사용해 뉴스 모니터링 전체 배치 사이클을 실행하라: 데이터 수집(백그라운드 collect.py, ~3분) → 미채점 export(중복 대표만) → 청크 분할 → 병렬 채점 에이전트(반드시 청크별 에이전트를 한 메시지에서 동시에 모두 띄워라, 단 model 오버라이드 없이 표준 컨텍스트로, 각 에이전트에 입력 id 개수와 출력 키 개수가 같은지 자가검증 후 저장하도록 지시) → 병합·커버리지 검증(누락 0, 누락분은 직접 보충 채점) → apply(점수 그룹 전파·게이팅·중복 물리제거) → 검증(미채점 0)·임시파일 정리. 프로젝트 루트: G:\내 드라이브\개인개발\news_curating (CWD, .venv\Scripts\python.exe). 디스크 여유를 먼저 확인하고 부족하면 ms-playwright·pip캐시·__pycache__를 정리하라. 완료 후 7일 중요뉴스 수를 카테고리별로 한 줄 요약하라.
