# News Design Review — 대시보드 디자인 품질 심사

동작하는 대시보드를 **냉정하게 평가하고 라운드마다 실제로 개선**하는 스킬. 제품 철학("빈도 최소화, 관심도 최대화")이 화면에 드러나야 한다 — 정말 중요한 알림이 시각적으로 가장 먼저 눈에 들어와야 한다.

## 계약 보존 (절대)
`src/web/`만 수정. 프론트가 읽는 필드명 불변: `importance_score`, `category`, `category_label`, `url`, `title`, `importance_reason`, `source`, `published_at`. 목록 응답은 `{items, total}`. 무프레임워크·무빌드 유지. **회귀 금지** — 직전 라운드보다 나빠지지 않는다.

## 평가 루브릭 (0~100 가중 합산)

| 축 | 가중 | 핵심 질문 |
|----|------|-----------|
| 정보 위계 | 25% | 발송 대상 알림이 최우선으로 보이는가? 점수·카테고리·발송여부의 시각 무게가 의미와 맞는가? |
| 간격·리듬 | 15% | 여백·정렬·그리드가 일관된가? |
| 컬러 시스템 | 15% | 팔레트가 절제·의미적인가? 카테고리/점수 색 구분이 명확한가? |
| 타이포 | 15% | 스케일·웨이트·행간이 위계를 만드는가? 한글 가독성은? |
| 일관성 | 10% | 컴포넌트가 규칙적인가? 일회성 인라인 스타일 남발 감점. |
| 접근성 | 10% | 대비 4.5:1+, 포커스 가시성, 시맨틱, 색 비의존 정보. |
| 반응형 | 10% | 모바일~데스크톱 깨짐 없는 재배치. |

## 라운드 절차
1. `_workspace/design_review_log.md`에서 이전 점수·미해결 항목 확인.
2. 라이브 대시보드를 **새로 로드** 후 데스크톱·모바일 스크린샷 평가.
3. 7축 점수 + 총점 산출, 임팩트 큰 결함 1~3개 선정.
4. `src/web/`에서 작고 안전하게 수정(회귀 없이).
5. 재로드·재스크린샷으로 개선 확인.
6. 로그에 라운드 섹션 append: 총점, 축별, 발견, 변경, 다음 후보.

## 흔한 결함 체크리스트
- 위계: 알림/일반 카드 구분이 약함, 점수 배지가 너무 작거나 색 의미 모호.
- 간격: 섹션 간 간격 불균일, 카드 내부 패딩 들쭉날쭉, 그리드 gap 불일치.
- 컬러: 카테고리 액센트가 좌측 보더/pill에 일관 적용 안 됨, 점수 색 임계 경계 모호.
- 타이포: 제목/본문/메타 위계 약함, 너무 작은 메타 텍스트 대비 부족.
- 일관성: 인라인 style 남발(토큰화 필요), 호버/포커스 상태 누락.
- 접근성: 회색 위 회색 텍스트 대비 미달, time/aria 누락, 링크 포커스 불가시.
- 반응형: 헤더 줄바꿈 깨짐, 설정 패널 모바일 오버플로우, 카드 1열 전환 누락.

## 수렴 처리
후반 라운드에 큰 결함이 없으면 churn 금지 — 마이크로 정제(정렬 픽셀, 대비 미세조정, 인터랙션 디테일)에 그치고 "수렴 단계"로 기록. 없는 문제를 지어내지 말 것.

## 검증
변경마다 라이브 재렌더로 확인. 픽셀 확인 불가 환경이면 명시하고 최소한 계약 필드 정적 점검. 테스트(`pytest`)는 프론트 변경과 무관하게 계속 그린이어야 한다.
